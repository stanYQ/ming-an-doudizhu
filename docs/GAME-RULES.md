## 二、牌值编码（供 CardEncoding.ts 实现）

### 2.1 整数编码方案（0–107）

```
普通牌：deck × 54 + suit × 13 + rank
  deck  ∈ {0, 1}          — 第一副 / 第二副
  suit  ∈ {0=♠, 1=♥, 2=♦, 3=♣}
  rank  ∈ {0=3, 1=4, 2=5, 3=6, 4=7, 5=8, 6=9,
            7=10, 8=J, 9=Q, 10=K, 11=A, 12=2}

小王：deck × 54 + 52
大王：deck × 54 + 53
```

**示例：**
- ♠3（第一副）= 0×54 + 0×13 + 0 = 0
- ♠3（第二副）= 1×54 + 0×13 + 0 = 54
- 小王（第一副）= 0×54 + 52 = 52
- 大王（第二副）= 1×54 + 53 = 107

### 2.2 compareValue（比较权重）

```
大王 = 17
小王 = 16
2   = 15
A   = 14
K   = 13
Q   = 12
J   = 11
10  = 10
9   = 9
...
3   = 3
```

同一点数两副牌的 compareValue 相同（两张 ♠3 权重都是 3）。

## 三、牌局完整流程

### 3.1 发牌流程

系统洗混 108 张牌后：
1. 随机选 1 张作为**明牌**，向所有玩家公示点数和花色
2. 将全部 108 张（含明牌）正常发牌：每人 21 张（共 105 张），剩余 3 张为底牌
3. 明牌会自然落入某位玩家的 21 张手牌中，该玩家成为地主
4. 地主拿走底牌（手牌变为 24 张）

> **实现说明**：明牌仅是「预先亮出的标记」，不单独抽出。发牌时按正常流程处理所有 108 张，明牌所在位置决定地主身份。总牌数始终为 108 = 21×5 + 3。

### 3.2 确定地主

1. 系统从 108 张中随机选 1 张作为明牌，向所有玩家公示其点数和花色
2. 洗牌，发牌：每人 21 张，剩余 3 张为底牌
3. 持有明牌的玩家自动成为**地主**（若两副牌都有此牌，持有明牌那张的玩家为地主）
4. 地主查看底牌，将 3 张底牌并入手牌（地主共 24 张）

### 3.3 选暗号牌

1. 地主从手牌中指定一张牌作为**暗号牌**
2. 合法选择：点数 3–10（rank 0–7），必须指定花色，共 32 种
3. 非法选择：J/Q/K/A/2/小王/大王（系统拦截，错误码 1001）
4. 系统寻找另一张相同具体花色的牌（两副牌共有 2 张同一花色同一点数）
5. 持有另一张的玩家为**暗队友**
6. **一挑四触发**：若两张暗号牌都在地主手中（地主既持有明牌那张，又持有另一副的同牌）

### 3.4 对局进行

1. 地主先出牌
2. 按开局固定方向（顺时针或逆时针，开局前确定，整局不变）依次行动
3. 首次打出暗号牌时，系统自动公开该玩家的身份（暗队友身份揭露）
4. 第一个出完手牌的玩家，其阵营获胜，进入结算

## 四、阵营规则

### 4.1 标准模式

| 阵营 | 成员 | 胜利条件 |
|------|------|---------|
| 地主阵营 | 地主 + 暗队友 | **地主或暗队友**任意一人最先出完手牌 |
| 平民阵营 | 其余 3 人 | 地主阵营中无人最先出完 |

### 4.2 一挑四模式（两张暗号牌都在地主手中）

| 阵营 | 成员 | 胜利条件 |
|------|------|---------|
| 地主阵营 | 地主 1 人 | 地主**本人**最先出完 |
| 平民阵营 | 其余 4 人 | 地主不是第一个出完 |

**一挑四胜负判定**：任意玩家出完手牌，游戏立即结束。地主不是第1名 → 平民阵营胜，无论是哪个平民先出完。

**标准模式暗队友胜负判定**：服务端内部始终持有暗队友身份（`hands` Map 和 `codeCardHolder` 字段），一旦任何玩家手牌数归零，立即按服务端内部状态查询其阵营。身份公开（`identity_reveal`）是客户端视觉效果，不影响服务端判定时机。

## 五、出牌规则

### 5.1 基本出牌规则

1. 地主先出牌，之后按固定方向依次行动
2. 轮到自己时：可以「出牌」或「不要（pass）」
3. **出牌**：必须打出合法牌型，且点数/大小大于上家出的同类牌型
4. **不要**：跳过本轮
5. 一轮中其余所有玩家全部「不要」时，最后出牌者获得新一轮**自由出牌权**（可出任意合法牌型）
6. 炸弹和王炸可压制任意普通牌型（不要求同类型）

### 5.2 压牌规则

| 情况 | 是否合法 |
|------|---------|
| 同类型 + 张数相同 + 点数更大 | ✓ |
| 炸弹压普通牌型 | ✓ |
| 王炸压任意牌（含炸弹） | ✓ |
| 同类型 + 张数不同 | ✗ |
| 普通牌型压炸弹 | ✗ |

## 六、牌型系统

### 6.1 合法牌型一览

| 牌型 | 张数 | 条件 | 示例 |
|------|------|------|------|
| 单张 | 1 | 任意 | ♠7 |
| 对子 | 2 | 同点数 × 2 | ♠5 ♥5 |
| 三张 | 3 | 同点数 × 3 | 8 8 8 |
| 三带一 | 4 | 三张 + 任意 1 张 | 9 9 9 + 4 |
| 三带二 | 5 | 三张 + 对子 | J J J + 6 6 |
| 顺子 | ≥5 | 连续单牌，不含 2 和王 | 3 4 5 6 7 |
| 连对 | ≥6（偶数）| ≥3 组连续对子，不含 2 和王 | 3 3 4 4 5 5 |
| 飞机 | ≥6 | ≥2 组连续三张，不含 2 和王 | 3 3 3 4 4 4 |
| 飞机带翅膀 | ≥8 | 飞机 + 等组数的附带（单张或对子） | 333 444 + 7 8 |
| 炸弹 | 4–8 | 同点数 4–8 张 | 8 8 8 8 |
| 王炸（小） | 2 | 双小王 | 小王 小王 |
| 王炸（大） | 2 | 双大王 | 大王 大王 |

**顺子/连对/飞机中的重复点数规则**：同一点数在顺子/连对/飞机中只能出现 1 次。

- `♠3(deck0) ♠3(deck1) ♥4 ♥5 ♦6 ♣7` → **不合法**（有 2 张 3）
- `♠3 ♥4 ♦5 ♣6 ♠7` → **合法**（每点数 1 张）

两副牌带来的重复牌对顺子类牌型无加成，多余的同点数牌必须单独使用。

**王牌合法组合完整表**：

| 组合 | 是否合法 | 说明 |
|------|---------|------|
| 小王 + 小王 | ✓ 王炸 | 唯一规则：双小王 |
| 大王 + 大王 | ✓ 王炸 | 唯一规则：双大王 |
| 小王 + 大王 | ✗ | GDD 明确禁止 |
| 小王 × 1 | ✓ 单张 | compareValue = 16 |
| 大王 × 1 | ✓ 单张 | compareValue = 17 |
| 小王 × 3 | ✗ | 不存在此牌型，必须拆开使用 |
| 大王 × 3 | ✗ | 不存在此牌型，必须拆开使用 |
| 4 张王任意组合 | ✗ | 不存在「四王炸」 |

> **实现要点**：玩家手里若有 3 张小王，只能选其中 2 张打王炸，第 3 张只能作为单张出。CardPatternEngine 解析时，王牌组只识别「恰好 2 张相同类型王」为王炸，其余情况均视为若干单张。

### 6.2 炸弹规则（双副牌扩展）

同一点数 4–8 张即为炸弹：

| 炸弹类型 | 张数 | 比较规则 |
|---------|------|---------|
| 四张炸 | 4 | 同档位比点数：3 < 4 < … < K < A < 2 |
| 五张炸 | 5 | 五张炸 > 任意四张炸 |
| 六张炸 | 6 | 六张炸 > 五张炸 |
| 七张炸 | 7 | 七张炸 > 六张炸 |
| 八张炸 | 8 | 八张炸 > 七张炸（普通炸弹最大）|

**炸弹比较逻辑**：先比张数（多者胜），张数相同再比点数。

**完整大小序列**：
```
双大王炸 > 双小王炸 > 八张炸 > 七张炸 > 六张炸 > 五张炸 > 四张炸 > 普通牌型
```

### 6.3 王炸规则

- **单张王**：大王（compareValue=17）和小王（compareValue=16）均可作为单张出，参与单张比较
- 双小王（2 张小王）= 王炸（JOKER_BOMB_SMALL），可压所有普通炸弹
- 双大王（2 张大王）= 王炸（JOKER_BOMB_BIG），可压双小王炸，无敌
- 小王 + 大王 = **不是王炸**（GDD 明确规定），也不是任何合法牌型
- 3 张及以上王的任意组合 = 非法
- 不存在「四王炸」

## 七、暗号牌规则

### 7.1 合法暗号牌

- 点数：3、4、5、6、7、8、9、10（rank 0–7）
- 花色：♠♥♦♣（suit 0–3）
- 合法组合：8 × 4 = **32 种**
- 非法：J/Q/K/A/2 和所有王

### 7.2 暗号牌的游戏效果

1. 地主指定具体花色+点数（如「红心 7」）
2. 系统寻找另一张「红心 7」的持有者 → 该玩家为**暗队友**
3. 暗队友身份对所有玩家隐藏，直到该玩家打出暗号牌
4. 打出暗号牌时：系统自动广播 `identity_reveal`，公开该玩家身份
5. **暗号牌本身是普通牌**，打出后按正常牌型参与出牌判断

### 7.3 一挑四触发条件

地主指定暗号牌后，若两张该花色+点数的牌**都在地主手中**，触发一挑四：
- 地主独自对抗其余 4 名平民
- 无暗队友
- 结算倍率 × 2

## 八、胜负判定

### 8.1 标准模式

```
触发条件：任意玩家手牌数变为 0
判定逻辑：
  if 出完者 == 地主 OR 出完者 == 暗队友:
      地主阵营胜
  else:
      平民阵营胜
```

注：服务端始终知道暗队友身份（内部状态），判定不依赖身份是否已公开。

### 8.2 一挑四模式

```
触发条件：任意玩家手牌数变为 0
判定逻辑：
  if 出完者 == 地主:
      地主胜
  else:
      平民阵营胜（地主负）
```

## 九、结算积分

### 9.1 基础公式

```
积分变动 = BaseScore × 总倍率 × 阵营系数

BaseScore（按段位）:
  青铜 = 100, 白银 = 120, 黄金 = 150
  铂金 = 180, 钻石 = 220, 王者 = 260

阵营系数:
  胜方 = +1
  负方 = -1

总倍率 = 模式倍率 × 炸弹累乘 × 王炸累乘
  模式倍率: 标准 × 1，一挑四 × 2
  每出一个炸弹: × 2（累乘）
  每出一个王炸: × 3（累乘）
  八张炸弹（该次炸弹）: × 4（替换当次的 × 2）
```

### 9.2 分配规则

| 对局结果 | 地主 | 暗队友 | 平民（每人）|
|---------|------|--------|------------|
| 标准·地主阵营胜 | +2份 | +2份 | -1份 |
| 标准·平民胜 | -2份 | -2份 | +4/3 份 |
| 一挑四·地主胜 | +8份 | — | -2份 |
| 一挑四·地主负 | -4份 | — | +1份 |

> **平民 +4/3 份说明**：地主-2 + 暗队友-2 = 4份，平摊给 3 名平民，每人得 4/3 ≈ 1.33 份。积分以浮点数存储，展示时四舍五入到整数。`score_delta = round(BaseScore × 总倍率 × 4/3)`

### 9.3 段位与积分区间

| 段位 | 积分区间 | 晋级要求 |
|------|---------|---------|
| 青铜 | 0–999 | 积分达 1000 |
| 白银 | 1000–1999 | 积分达 2000 |
| 黄金 | 2000–3499 | 积分达 3500 |
| 铂金 | 3500–4999 | 积分达 5000 |
| 钻石 | 5000–7499 | 积分达 7500 |
| 王者 | 7500+ | 赛季榜单 |

## 十、超时与托管

### 10.1 出牌超时

- 出牌限时：**30 秒**
- 超时处理：服务端自动 pass（不要）
- 连续 3 次超时 → 进入**托管模式**

### 10.2 托管模式

托管 AI 逻辑：
- **自由出牌轮**（上一轮自己赢得出牌权）：出手牌中 compareValue 最小的**单张**
- **跟牌轮**（上家出了牌，需要压或 pass）：直接 **pass**

> **实现要点**：托管 AI 刻意保守，优先 pass，只在必须出牌时出最小单张。目的是不干扰对局结果，让真实玩家重连后能继续。

### 10.3 断线重连

- 保留席位时长：**60 秒**
- 重连后恢复内容：
  1. 完整 GameState（Schema diff 自动同步）
  2. 私密手牌（服务端重发 `your_hand`）
  3. 当前出牌轮信息（`turn_change` 重发）

## 十一、匹配规则

### 11.1 快速匹配

- 按段位区间分桶（分桶边界与段位表一致）
- 匹配超时 **30 秒**未满 5 人 → AI 补位开局
- 匹配队列：Redis LIST，键名 `match:queue:{tier}`

**匹配容忍度**：
- 0–15 秒：仅同段位
- 15–30 秒：扩大到相邻 ±1 段位（青铜可匹配白银，白银可匹配青铜/黄金）
- 30 秒超时：AI 补位，不再等待

### 11.2 好友房

- 创建生成 **6 位房间码**，分享后 5 人进房即开局
- 消耗道具：房卡
- 游戏机制与快速匹配完全相同（暗号队友机制不变）
