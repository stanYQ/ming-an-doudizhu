# 明暗斗地主 UI 设计需求单 v1.1

**文档类型**: 设计需求单（交设计师/前端）  
**目标平台**: 微信小程序（主）/ H5（次）  
**设计基准分辨率**: 1280 × 720（逻辑坐标，横屏锁定）  
**背景图实际尺寸**: 1600 × 720（两侧各多 160px，防宽屏黑边）  
**适配模式**: FitHeight（Cocos Creator 横屏标准，高度撑满，宽度居中裁切）  
**安全区**: UI 元素约束在中央 1280×720 区域内；四边额外留 40px 避开刘海/Home 条  
**版本**: v1.1 | 日期: 2026-06-23  

---

## 一、设计语言总纲

| 维度 | 要求 |
|------|------|
| 风格 | 中式桌游 · 暗调 · 轻奢（深色牌桌 + 金色描边） |
| 主色 | 深绿 `#1A3A2A`（牌桌绒面）/ 深褐 `#2C1A0E`（边框） |
| 强调色 | 金黄 `#D4A843`（按钮高亮 / 地主标识）/ 红色 `#C0392B`（炸弹 / 胜利） |
| 字体 | 标题: 黑体加粗；正文: 苹方/思源黑体；数字: 等宽字体 |
| 卡牌 | 双副牌，花色需明显区分（♠♣ 黑色，♥♦ 红色），牌背统一深色花纹 |
| 动效 | 发牌滑入 200ms；出牌飞出 150ms；结算淡入 300ms；禁用重量级粒子 |
| 最小点击区 | 44 × 44 pt（微信小程序规范） |

---

## 二、场景清单与页面结构

```
LaunchScene      启动页（资源加载 + 登录）
  └── HallScene  大厅（玩家信息 + 匹配入口）
        └── MatchView（弹层）  快速匹配等待 / 好友房等待室
  └── GameScene  游戏桌
        ├── PlayerSeat × 5    5个席位
        ├── HandCardView      本人手牌区（底部）
        ├── PlayZone          出牌区（中心桌面）
        ├── CodeCardSelector  暗号牌弹窗（仅地主）
        ├── DoublingView      加倍面板（全员）
        └── SettlementView    结算界面（全屏遮罩）
```

### 3.1 LaunchScene — 启动页

**功能**: 资源加载进度 + Stub 登录过渡  
**显示时长**: 加载完成后自动跳转（不超过 10 秒）

| 元素 | 规格 |
|------|------|
| 背景 | 全屏游戏主视觉图（牌桌俯视角 + 5张暗牌）|
| 游戏 Logo | 「明暗斗地主」竖排书法字体，居中，W240 |
| 加载进度条 | 底部居中，W400 × H8，金色填充 + 深色底 |
| 进度文字 | 进度条下方 8px，「加载中… XX%」，14px 灰白 |
| 错误提示 | 红色文字居中，下方「重试」按钮（默认隐藏）|

**状态**:
- 加载中：进度条动画
- 加载失败：显示错误文字 + 重试按钮

**功能**: 展示玩家信息、进入匹配

**布局（横屏 1280×720）**:

| 元素 | 规格 |

| 按钮禁用态 | 透明度 40%，不可点击 |

### 3.3 MatchView — 匹配弹层

弹层覆盖大厅，半透明黑色蒙层（opacity 80%）

#### 3.3a 快速匹配等待

| 元素 | 规格 |
|------|------|
| 标题 | 「快速匹配」20px 白色 居中 |
| 人数进度 | 「X/5 人已加入」24px 金色 加粗 |
| AI 补位倒计时区 | 「XX 秒后 AI 补位」14px 灰白；归零显示「AI 补位中…」；满员隐藏 |
| 等待动画 | 3个圆点循环跳动（CSS/Tween），颜色金色 |
| 取消按钮 | W200×H48，描边风格，文字「取消匹配」；游戏开始后禁用变灰 |

**状态变化**:
- `readyCount < 5`：显示倒计时区
- `aiSeconds = 0`：倒计时区显示「AI 补位中…」
- `readyCount = 5`：隐藏倒计时区，显示「即将开始…」

#### 3.3b 好友房等待室

| 元素 | 规格 |
|------|------|
| 标题 | 「好友房」20px 白色 居中 |
| 房间码 | 大号数字 32px 金色，格式「房间码: XXXXXX」；复制按钮在右侧 |
| 席位列表 | 5行，每行显示头像占位圆 + 昵称（空席显示「等待加入…」灰色）|
| 开始按钮 | 仅房主可见，W240×H56，金色背景；人数 < 2 时禁用 |
| 等待提示 | 非房主显示「等待房主开始…」灰色斜体 |
| 分享按钮 | 「分享给好友」，图标 + 文字，描边风格 |
| 错误提示 | 红色文字，错误码对应中文（见协议 §9）|

**整体布局**:

#### 3.4a PlayerSeat — 玩家席位 × 5

每个席位包含：

| 元素 | 规格 |
|------|------|
| 头像框 | 圆形 W56×H56；当前出牌者金色光圈动画 |
| 昵称 | 12px 白色，头像下方 4px，最多 6 字截断 |
| 手牌数 | 「剩X张」12px 灰色，昵称下方 |
| 计时圆环 | 头像外圈，颜色从绿→黄→红（30s 倒计时），非当前出牌者隐藏 |
| 身份标签 | 「地主」金色 /「搭档」橙色（揭露后显示，默认隐藏）|
| 出完标识 | 「出完」绿色 badge，手牌数归 0 时显示 |
| AI 标识 | 「AI」灰色小标，`isAI=true` 时显示 |

**席位位置（5人椭圆桌布局）**:
- 席位0（本人）: 底部居中
- 席位1: 左侧中间
- 席位2: 左上
- 席位3: 右上
- 席位4: 右侧中间

#### 3.4b HandCardView — 本人手牌

| 元素 | 规格 |
|------|------|
| 手牌区 | 底部横排，最多 24 张（地主含底牌）|
| 单张牌 | W52×H78，圆角 6，白色底；花色+点数居中显示 |
| 选中态 | 牌向上偏移 16px，金色描边 2px |
| 牌型提示 | 手牌区上方，「单张 / 对子 / 顺子…」14px；非法选择显示「请选择合法牌型」灰色 |
| 底牌展示 | 地主 `landlord_select` 阶段，在手牌区上方单独一行展示 3 张底牌，标注「底牌」|
| 禁用态 | 非本人回合，所有牌透明度 60%，不可点击 |
| 提示高亮 | 收到 `hint` 消息后，推荐牌蓝色描边 2px 闪烁 2 次 |

**牌的视觉规格**:
- 王牌（大王）: 红底金字「大王」
- 王牌（小王）: 黑底白字「小王」
- 普通牌: 白底，♠♣ 黑色，♥♦ 红色

#### 3.4c PlayZone — 出牌区（桌面中心）

| 元素 | 规格 |
|------|------|
| 上家出牌展示 | 最近一次出牌，居中横排，牌面朝上；自由回合时清空 |
| 出牌者昵称 | 出牌区上方 8px，「XX 出了：」12px 灰色 |
| PASS 标识 | 其他玩家不要时，席位旁显示「不要」气泡 500ms 后消失 |
| 操作按钮行 | 「出牌」「不要」「提示」三个按钮，底部居中 |
| 出牌按钮 | W120×H48，金色，非本人回合禁用 |
| 不要按钮 | W120×H48，描边风格，非本人回合 / 自由轮禁用 |
| 提示按钮 | W120×H48，描边风格，本人回合可用 |
| 错误提示 | 按钮上方，红色文字，「牌型不合法 / 压不过上家」，2s 后消失 |
| 出牌倒计时 | 席位计时圆环（见 PlayerSeat），此处不重复 |

| 搭档标识 | 「搭档」橙色 badge |

## 四、全局 UI 规范

### 4.1 按钮状态

| 状态 | 视觉 |
|------|------|
| 正常 | 100% 不透明，正常颜色 |
| Hover（H5）| 亮度 +10% |
| 按下 | 亮度 -10%，轻微缩放 0.96 |
| 禁用 | 透明度 40%，cursor not-allowed |
| 加载中 | 显示旋转 Spinner，文字隐藏 |

### 4.2 Toast / 错误提示

- 位置: 屏幕顶部居中，距顶 80px
- 尺寸: 自适应宽，高 44px，圆角 22，内边距 16×12
- 颜色: 错误红 `#C0392B`，成功绿 `#27AE60`，普通深灰 `#333`
- 动效: 淡入 150ms → 停留 2s → 淡出 150ms

### 4.3 网络状态提示

- 断线: 顶部红色横幅「网络已断开，尝试重连…」
- 重连中: 黄色横幅 + 旋转图标
- 重连成功: 绿色横幅 2s 后消失

### 4.4 加载遮罩

- 全屏半透明黑，中心旋转 Spinner（金色，W48）
- 用于: joinRoom 等待、场景切换

## 五、屏幕适配规范

### 5.1 分辨率与适配模式

| 项目 | 规格 | 说明 |
|------|------|------|
| 设计基准 | 1280 × 720 | 逻辑坐标，所有 UI 元素在此范围内定位 |
| 背景图尺寸 | **1600 × 720** | 两侧各扩 160px，19.5:9 / 20:9 宽屏无黑边 |
| 适配模式 | **FitHeight** | 高度撑满屏幕，宽度等比缩放后居中；超出部分裁切 |
| UI 安全区 | 水平方向 ±640（即 1280 宽内）| 超出 ±640 的区域在部分机型上不可见，禁止放置关键元素 |
| 刘海/Home 条 | 四边各留 40px | 通过 Cocos SafeAreaWidget 自动适配 |

### 5.2 主流机型覆盖验证

| 机型类别 | 屏幕比例 | FitHeight 效果 |
|---------|---------|--------------|
| 旧 Android（720p）| 16:9 | 设计区完全填满，无黑边 |
| iPhone 14/15 | 19.5:9 | 背景两侧延展，UI 居中不裁切 |
| 三星 S 系列 | 20:9 | 背景两侧延展，UI 居中不裁切 |
| iPad 横屏 | 4:3 | 上下出现少量黑边（可接受）|

> **设计师注意**：背景图必须是 1600×720，左右各 160px 为延伸区（可放渐变/纹理，不放文字和按钮）。

## 六、微信小程序适配要求

| 项目 | 要求 |
|------|------|
| 主包体积 | ≤ 2MB（bundle 分包：hall / game / common）|
| 安全区 | 使用 Cocos SafeAreaWidget（已实现于 `core/SafeAreaWidget.ts`）|
| 字体 | 不引入外部字体，使用系统默认（苹方 / 思源）|
| 图片格式 | WebP 优先，PNG 降级；单图 ≤ 200KB |
| 图集合并 | 卡牌图集 / UI 图集各一张，减少 DrawCall |
| 震动反馈 | 出炸弹时调用 `wx.vibrateShort()`（需用户授权）|
