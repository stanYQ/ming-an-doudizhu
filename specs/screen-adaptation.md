# Spec: 横屏适配基础配置

**任务 ID**: TASK-016  
**目标模块**: client  
**优先级**: P2  
**状态**: ready  
**前置依赖**: 无（所有 P2 UI 任务的基础，应最先完成）

---

## 背景

来源：GDD v1.0 第八章（8.1 美术规范）+ TDD v1.0 第一章（技术栈：Cocos Creator 3.8 + FairyGUI + oops-framework）。游戏为横屏（Landscape）方向，目标平台为微信小程序、H5、Android、iOS 四端。不同设备的宽高比从 16:9 到 21:9 不等，iOS 和 Android 存在安全区（刘海、打孔屏、Home 条），需在开发初期建立统一的适配基线，避免各 UI 组件各自处理适配导致不一致。

**本 spec 定义「适配要求」（WHAT），实现方案由 dev 在 Cocos Creator 3.8 + FairyGUI + oops-framework 框架能力范围内决定（HOW）。**

## 验收标准

### 方向锁定

- AC-1: 游戏全程强制横屏（Landscape），不随设备旋转自动切换为竖屏
- AC-2: 微信小程序、H5、Android、iOS 四端均有效锁定横屏

### 设计分辨率

- AC-3: 设计基准分辨率为 **1280 × 720**（来自 GDD 8.1），所有 UI 元素以此为参照设计
- AC-4: 实际设备分辨率偏高/偏低时，UI 整体等比缩放，不出现拉伸变形

### 宽高比适配

- AC-5: 在宽高比 16:9（如 1280×720）下，UI 布局无遮挡、无溢出
- AC-6: 在宽高比 18:9（如 2160×1080）下，两侧出现留黑或背景延伸，核心交互区不受影响
- AC-7: 在宽高比 19.5:9（如 2532×1170，iPhone 12/13/14）下，核心交互区无遮挡
- AC-8: 在宽高比 21:9（如 2560×1080）下，核心交互区无遮挡

### 安全区（Safe Area）

- AC-9: iOS 刘海机型（iPhone X 及以后）的刘海区域和底部 Home 条区域内，不放置任何可点击按钮
- AC-10: Android 打孔屏（摄像头区域）不遮挡关键 UI 元素
- AC-11: 底部手牌区、出牌/不要按钮、右上角设置按钮均在安全区内

### 多端构建验证

- AC-12: 微信开发者工具预览（iPhone 6/X/14 Pro Max 机型）均通过 AC-5 至 AC-11
- AC-13: 浏览器 H5（Chrome DevTools 设备模拟，选 16:9 / 18:9 / 19.5:9）通过 AC-5 至 AC-8

## 核心交互区定义（供所有 P2 UI spec 参照）

横屏下各区域的安全范围（以 1280×720 为基准）：

```
│  │  顶部信息区（房间号、明牌、底牌）     [设置按钮]  │  │
│  │                                                   │  │

```

## 约束

- 技术栈：Cocos Creator 3.8 + FairyGUI + oops-framework（TDD 第一章，不得引入其他适配库）
- 适配方案的具体实现（Canvas 设计分辨率设置、FairyGUI UIContentScaler 参数、安全区 inset 读取方式）由 dev 决定
- 所有 P2 UI 组件均须在此适配基线建立后再开始实现；**TASK-016 是 P2 最高优先级任务**
- 背景图须能覆盖超宽屏两侧延伸区域，美术资源按 1920×1080 以上输出（超出设计分辨率的裁切余量）

## 不在范围内

- 竖屏模式（明确不支持）
- iPad 适配（P3，宽高比特殊处理）
- 折叠屏动态切换（P4）

## 测试要求

- 本 spec 无单元测试（配置类任务）
- 验收方式：在微信开发者工具 + Chrome DevTools 中逐一截图验证 AC-5 至 AC-13
- 检查清单：每个 P2 UI 组件完成后，server-dev / client-dev 在对应 spec 的「测试要求」里补充「横屏适配已验证 ✓」
